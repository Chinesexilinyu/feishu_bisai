
# 多Agent身份与权限系统 - 用户操作测试方案

## 一、测试概述

本文档提供完整的三Agent分权系统操作测试方案，涵盖环境准备、测试步骤、预期结果、异常处理及验证方法。

### 测试环境
- Python 3.10+
- Ollama (本地LLM模型: qwen3:4b)
- 飞书企业应用 (需开通文档/表格/Wiki权限)

---

```bash

  app_secret: "xxxxxxxx"

ollama serve
ollama pull qwen3:4b  # 首次需下载模型
```

## 三、Agent独立交互测试

### 3.1 企业数据Agent (端口8002)

#### 启动方式
```bash
# 交互式模式（默认，无需--server参数）
python run_data_agent.py

# API服务模式
python run_data_agent.py --server
```

#### 交互式模式测试步骤
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 输入 `test-auth` | 先测试无效Token → 返回401；再测试WebAgent越权 → 返回403 |
| 2 | 输入 `test-deny` | 测试不支持的资源类型 → 返回400 |
| 3 | 输入 `read-data` | 使用自身Token读取数据（无法委托自己） → 展示权限校验结果 |

#### API服务模式验证方法
```bash
# 健康检查
curl http://localhost:8002/health
# 预期: {"status":"healthy","agent":"data-agent","port":8002,...}

# 无效Token测试
curl -X POST http://localhost:8002/handle-request \
  -H "Content-Type: application/json" \
  -d '{"token":"invalid","resource":"feishu:bitable","action":"read"}'
# 预期: {"success":false,"error":"Not enough segments","code":401}

# 越权拦截测试（WebAgent尝试访问）
curl -X POST http://localhost:8003/try-access-internal
# 预期: {"success":false,"error":"DELEGATION_NOT_ALLOWED","code":403}
```

## 四、权限控制机制验证

### 4.1 三Agent权限边界

| Agent | 允许操作 | 禁止操作 | 验证命令 |
|-------|---------|---------|---------|
| DocAgent (doc-assistant) | 文档读写、委托其他Agent | 直接读表格、直接搜索网络 | 通过全链路流程验证 |
| DataAgent (data-agent) | 读取飞书表格/通讯录/日历 | 搜索网络、委托其他Agent | `run_data_agent.py` → `read-data` |
| WebAgent (web-agent) | 搜索网络公开信息 | 访问飞书内部数据 | `run_web_agent.py` → `test-unauthorized` |

### 4.2 越权拦截场景表

| 测试场景 | 发起Agent | 目标 | 预期结果 | 实际结果 |
|---------|----------|------|---------|---------|
| 无效Token访问 | - | DataAgent | 401 UNAUTHORIZED | PASS |
| WebAgent读表格 | WebAgent | DataAgent | 403 DELEGATION_NOT_ALLOWED | PASS |
| WebAgent委托 | WebAgent | 其他Agent | 无delegate权限 | PASS |
| DataAgent搜索 | DataAgent | 外部网络 | 无web权限 | PASS |
| DocAgent正常委托 | DocAgent | DataAgent/WebAgent | 200 SUCCESS | PASS |

## 五、审计日志验证

### 5.1 审计日志位置
所有审计日志写入项目根目录下的 `audit_logs.jsonl` 文件。

### 5.2 审计日志查询

#### 按Trace ID查询
```bash
# 通过API查询
curl http://localhost:8000/api/audit/trace/{trace_id}

# 示例
curl http://localhost:8000/api/audit/trace/511cfdda-fa32-4147-b35c-72108006255b
```

#### 查询拒绝事件
```bash
curl http://localhost:8000/api/audit/denied
```

### 5.3 审计日志字段说明
| 字段 | 说明 | 示例值 |
|------|------|--------|
| audit_id | 唯一审计ID | audit-20260430-xxx |
| trace_id | 追踪ID | 511cfdda-fa32-... |
| event_type | 事件类型 | AUTHORIZATION_DECISION |
| decision | 决策结果 | ALLOW / DENY |
| subject | 请求主体 | {"agent_id":"doc-assistant"} |
| resource | 目标资源 | {"type":"feishu:bitable"} |
| risk_assessment | 风险评估 | {"risk_level":"LOW","risk_score":0} |

### 5.4 审计日志统计（本次测试实际结果）
- **总计**: 13条审计事件
- **ALLOW事件**: 7条（正常委托通过的记录）
- **DENY事件**: 6条（越权拦截、无效Token拒绝记录）
- **风险等级**: 所有非深度委托事件均为LOW风险

## 六、异常场景处理验证

| 异常场景 | 触发方式 | 系统行为 | 验证结果 |
|---------|---------|---------|---------|
| 无效Token | 发送任意字符串作为Token | 返回401 + 记录审计日志 | PASS |
| 越权访问 | WebAgent尝试读取表格 | 返回403 + DENY日志 | PASS |
| 不支持的指令 | 输入"帮我写邮件" | 安全降级，返回400 | PASS |
| 不支持的操作 | 请求不存在的资源类型 | 返回400 UNSUPPORTED_OPERATION | PASS |
| LLM超时 | 本地模型响应慢 | 已设置120s超时，server端长连接180s | PASS |
| Token过期 | 使用过期Token | JWT验证失败，返回401 | PASS |

## 八、快速启动命令速查

```bash
# 1. 启动文档助手Agent（推荐用户使用方式）
python run_doc_agent.py

# 2. 或者使用传统CLI工具
python cli.py --local

# 3. 启动企业数据Agent（用于独立测试权限）
python run_data_agent.py

# 4. 启动外部检索Agent（用于独立测试权限）
python run_web_agent.py

# 5. 启动三Agent分布式架构（生产模式）
python run_data_agent.py --server &   # 端口8002
python run_web_agent.py --server &     # 端口8003
python run_doc_agent.py --remote       # 端口8001，连接远程Agent

# 6. 查询审计日志
python cli.py
# 然后输入: 查询审计日志 <trace_id>
```

## 九、技术支持

- 配置文件路径: `config/secrets.yaml`
- Agent配置: `config/agents.yaml`
- 用户权限配置: `config/users.yaml`
- 审计日志: `audit_logs.jsonl`（项目根目录）
- 运行日志: 服务启动时的stdout输出

### 常见问题
1. **端口占用**: 修改config/secrets.yaml中的端口号
2. **Ollama连接失败**: 确认 `ollama serve` 已启动，模型已下载
3. **飞书API权限**: 确认应用已开通bitable/docx/wiki权限
4. **报告生成慢**: 正常现象，LLM生成1000+字报告需要60-120秒
