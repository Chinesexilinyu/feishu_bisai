# 审计日志字段说明
## 基本信息
| 项 | 值 |
|----|----|
| 存储格式 | JSON Lines (JSONL) |
| 存储位置 | 项目根目录`audit_logs.jsonl` |
| 记录触发时机 | 所有授权决策、任务状态变更、异常事件发生时 |
| 保留期限 | 永久保留（可配置定期归档） |
| 支持查询维度 | Trace ID、时间范围、决策结果、Agent ID、风险等级 |
| 支持导出格式 | JSON、CSV |

## 字段详细说明
| 字段名称 | 数据类型 | 触发条件 | 说明 |
|----------|----------|----------|------|
| audit_id | String | 所有事件 | 唯一审计ID，格式：`audit-{事件类型缩写}-{决策结果}-{时间戳}-{8位UUID}`，如：`audit-TASKINIT-ALLOW-20260501120000-abc123ef` |
| timestamp | String | 所有事件 | 事件发生时间，ISO 8601格式（UTC时间），如：`2026-05-01T12:00:00.000000` |
| trace_id | String | 所有事件 | 全链路Trace ID，UUID v4格式，同一任务所有事件共用同一个ID |
| event_type | String | 所有事件 | 事件类型：<br>- `TASK_INITIATED`：任务启动<br>- `TASK_EXECUTION`：任务执行<br>- `TASK_COMPLETED`：任务完成<br>- `TASK_FAILED`：任务失败<br>- `AUTHORIZATION_DECISION`：授权决策<br>- `AGENT_UNAVAILABLE`：Agent不可用 |
| event_category | String | 所有事件 | 事件分类，固定值：`A2A_AUTH`（Agent-to-Agent认证） |
| decision | String | 所有事件 | 决策结果：`ALLOW`（允许）/`DENY`（拒绝） |
| subject | Object | 所有事件 | 操作主体信息，包含：<br>- `agent_id`：Agent ID<br>- `agent_name`：Agent名称<br>- `agent_type`：Agent类型 |
| delegation_context | Object | 委托场景 | 委托上下文，包含委托用户信息、委托链深度等 |
| resource | Object | 所有事件 | 操作资源信息，包含：<br>- `type`：资源类型<br>- `action`：操作类型<br>- `resource_id`：资源ID（可选） |
| authorization | Object | 所有事件 | 授权信息，包含：<br>- `requested_capability`：请求的权限<br>- `reason`：决策原因<br>- `error_code`：错误码（可选）：<br>&nbsp;&nbsp;`0`：正常<br>&nbsp;&nbsp;`40302`：越权访问<br>&nbsp;&nbsp;`50301`：服务不可用 |
| risk_assessment | Object | 所有事件 | 风险评估结果，包含：<br>- `risk_level`：风险等级：`LOW`/`MEDIUM`/`HIGH`<br>- `risk_score`：风险分数（0~100）<br>- `risk_factors`：风险因子列表 |

## 记录触发条件
| 事件类型 | 触发时机 | 决策结果 | 错误码 |
|----------|----------|----------|--------|
| TASK_INITIATED | 用户发起任务请求时 | ALWAYS ALLOW | 0 |
| AUTHORIZATION_DECISION | Agent执行权限检查时 | ALLOW/DENY | 0/40302 |
| AGENT_UNAVAILABLE | 依赖的Agent服务未启动时 | ALWAYS DENY | 50301 |
| TASK_EXECUTION | 任务执行关键节点时 | ALLOW | 0 |
| TASK_COMPLETED | 任务执行成功完成时 | ALLOW | 0 |
| TASK_FAILED | 任务执行失败时 | DENY | 对应错误码 |

## 存储策略
1. 采用追加写入模式，日志一旦写入不可修改、不可删除
2. 按行存储JSON格式，便于流式查询和批量处理
3. 支持按时间自动分片存储（可配置）
4. 定期备份到对象存储，防止数据丢失
5. 支持冷热数据分离，历史数据归档后仍可查询
